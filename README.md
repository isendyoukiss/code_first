一、项目概述

Multi-Agent Content Ops 是一个基于浏览器端运行的多智能体协同内容运营自动化平台。系统模拟了一个完整的内容生产流水线，由 6 个专业化 AI Agent 各司其职、协同工作，实现从内容策划、撰写、SEO 优化、质量审核、多平台发布到数据追踪的全流程自动化。


项目以单个 HTML 文件交付，零依赖、零配置，打开即用。

三、核心模块详解

属性	说明
类型	发布/订阅（Pub/Sub）模式
职责	Agent 间解耦通信的基础设施
支持事件	agent:status、agent:progress、agent:message、log
特性	支持订阅返回取消函数，便于生命周期管理

3.2 Orchestrator — 编排引擎

编排器是系统的大脑，负责：


工作流创建：接收用户输入，实例化 ContentItem，注入流水线
顺序调度：按 Planner → Writer → SEO → Reviewer → Publisher → Analytics 顺序依次调用 Agent
状态管理：维护全局内容队列、统计数据、运行状态
异常处理：捕获流水线异常，防止系统崩溃
性能统计：累计产出量、通过率、平均耗时、协作次数

3.3 ContentItem — 内容实体模型

每个内容项是一个独立的状态机，经历 6 个阶段流转：

planning → writing → seo → review → publishing → published
  (策划)    (撰写)   (优化)  (审核)    (发布)      (完成)
                                    ↺ (未通过则留在 review)

字段	类型	说明
id	string	唯一标识符，自增生成
topic	string	内容主题
type	enum	article / social / newsletter / video_script / whitepaper
platform	enum	wechat / zhihu / weibo / douyin / all
keywords	string[]	关键词列表
stage	string	当前所在阶段
content	string	正文内容（由 Writer 生成）
seoScore	number	SEO 评分
reviewScore	number	审核综合评分
reviewComments	string[]	审核修改意见
timeline	array	全生命周期事件记录
metadata	object	大纲、SEO 分析、发布信息、表现数据等

四、六大 Agent 角色定义

4.1 Planner — 内容策划 Agent

角色: 内容策划
颜色: #3b82f6 (蓝色)
图标: 📋
输入: 用户提交的主题、类型、关键词
输出: 结构化大纲 + 受众画像 + 调性定义

职责：

分析内容主题的核心价值与切入点
根据内容类型自动生成对应大纲结构（5 种类型各有专属模板）
确定目标受众与内容调性
将大纲传递给 Writer Agent

4.2 Writer — 内容创作 Agent

角色: 内容创作
颜色: #f0a500 (琥珀色)
图标: ✍️
输入: 大纲结构 + 主题 + 类型
输出: 完整正文内容 + 字数统计 + 阅读时长

职责：

接收 Planner 传递的大纲
逐章节撰写内容，实时报告进度
统计字数与预估阅读时长
完成后将初稿传递给 SEO Agent

处理流程：

1.读取大纲中的章节列表
2.循环遍历每个章节，生成对应段落
3.每完成一个章节，通过 EventBus 广播进度
4.汇总全文，计算元数据
5.向 SEO Agent 发送协作消息

4.3 SEO — 搜索优化 Agent

角色: 搜索优化
颜色: #8b5cf6 (紫色)
图标: 🔍
输入: 完整正文 + 关键词列表
输出: SEO 分析报告 + 优化后内容

职责：

分析标题 SEO 评分（80-100 分区间）
计算关键词密度（1.5%-3.5%）
评估内容可读性评分
生成 Meta Description
统计内链建议数量
生成推荐标签集
在正文前插入摘要信息
将优化后内容传递给 Reviewer Agent

4.4 Reviewer — 质量审核 Agent

角色: 质量审核
颜色: #ff4757 (珊瑚红)
图标: 🔎
输入: SEO 优化后的内容
输出: 审核评分 + 修改意见 + 通过/驳回决定

职责：

对内容进行四维度评分
综合计算审核总分
决定是否通过（≥75 分通过）
未通过时生成具体修改意见
将通过的内容传递给 Publisher Agent

4.5 Publisher — 内容发布 Agent

角色: 内容发布
颜色: #00d4aa (青色)
图标: 🚀
输入: 审核通过的内容
输出: 发布状态 + 平台 URL

职责：

根据目标平台准备发布配置
模拟推送到对应平台
记录发布时间与 URL
通知 Analytics Agent 开始追踪

4.6 Analytics — 数据分析 Agent

角色: 数据分析
颜色: #06b6d4 (天蓝色)
图标: 📊
输入: 已发布内容
输出: 初始表现数据

职责：

追踪已发布内容的表现数据
采集曝光量、点击量、分享数、平均阅读时长
为运营决策提供数据支撑

五、Agent 间通信协议

所有 Agent 间通信通过 EventBus 进行，遵循以下消息格式

{
  from: string,      // 发送方 Agent 名称
  to: string,        // 接收方 Agent 名称
  message: string,   // 消息内容
  time: number       // 时间戳
}

通信流程图

Planner ──"大纲已生成，请撰写内容"──→ Writer
Writer  ──"初稿完成，请SEO优化"────→ SEO
SEO     ──"优化完成，请审核"────────→ Reviewer
Reviewer──"审核通过，可以发布"──────→ Publisher
Publisher──"已发布，请追踪数据"─────→ Analytics

6.2 各面板功能

Agent Cards Row

每个 Agent 一张卡片，显示名称、角色、状态徽章
完成数与消息发送数统计
实时进度条动画
工作中状态有脉冲呼吸灯效果
点击可查看 Agent 详情

内容流水线面板

顶部 6 个阶段标签可点击筛选
显示每个阶段的内容数量
内容列表按时间倒序排列
选中项高亮并同步详情面板
状态标签使用阶段对应色

Agent 通信日志

实时滚动显示所有 Agent 活动
时间戳 + Agent 名称 + 消息内容
不同 Agent 使用不同颜色标识
鼠标悬停高亮
最多保留 200 条记录

内容详情面板

内容标题 + 状态徽章 + 基本信息
正文预览（Markdown 渲染）
SEO 四维分析卡片
审核评分与修改意见
表现数据四宫格
处理时间线（可视化节点）

统计卡片

今日产出数量
审核通过率（目标 ≥85%）
平均处理时长
Agent 跨协作消息次数
